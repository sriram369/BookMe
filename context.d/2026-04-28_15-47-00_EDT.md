# BookMe Context Checkpoint

Timestamp: 2026-04-28 15:47 EDT

## Product

BookMe is an India-focused AI front desk platform for small and medium hotels. The public site sells the product. Hotel owners sign in or sign up, complete onboarding, connect their reservation source, and get a hotel-specific dashboard and guest-facing demo site. Guests can book, check in, and check out through a conversational front desk.

Current demo client: Sriram Hotel.

## Core Architecture

- Next.js 14 App Router application.
- Tailwind UI with the cinematic BookMe landing page and cleaner static hotel demo/admin surfaces.
- NextAuth v4 OAuth shell for Google/GitHub sign in.
- Supabase/Postgres persistence for BookMe users and hotel onboarding records.
- Google Sheets connector for hotel reservation/inventory data.
- OpenRouter agent wrapper for the demo LLM layer, defaulting to `openrouter/free`.
- Local JSON fallback only for local development when hosted production credentials are not configured.

## Main Routes

- `/` public BookMe landing page.
- `/signup` and `/signin` OAuth/local-demo entry pages.
- `/onboarding` hotel owner onboarding flow.
- `/demo?hotel=<slug>` guest-facing hotel demo.
- `/admin?hotel=<slug>` hotel owner admin dashboard.
- `/api/config` safe environment/config status.
- `/api/connectors` Google Sheets connector health/init endpoint.
- `/api/hotels` list/create hotel configs.
- `/api/hotels/[slug]` fetch one hotel dashboard, returns 404 for unknown slugs.
- `/api/agent` front-desk LLM endpoint.

## Important Implementation Notes

- `lib/db/supabase.ts` only allows server-side Supabase writes with `SUPABASE_SERVICE_ROLE_KEY`.
- `lib/auth/user-store.ts` persists users to Supabase when configured and only falls back to local JSON in local development.
- `lib/hotel/config-store.ts` persists hotel configs to Supabase when configured and only falls back to local JSON in local development.
- `middleware.ts` protects `/admin`, `/onboarding`, and `/solution` only when OAuth providers are configured. Without OAuth env vars, local demo mode stays open.
- `lib/hotel/tools.ts` uses Google Sheets when connector health is `ok`, otherwise falls back to demo data.
- Unknown hotel slugs correctly return 404 in API and Next not-found behavior in pages.
